En esta aplicación se puede consultar los correos desde el API JSONPlaceholder. De la misma manera, se puede crear un nuevo correo, eliminar uno existente y consultar detalle de un correo seleccionado.

Es importante destacar que el API utilizado NO soporta la persitencia real de los datos, por lo tanto, la aplicación hace el llamado de los métodos POST, GET y DELETE, pero sus datos solo cambian visualmente (caso eliminar), ya que una vez se recargue el componente que muestra la lista de los correos, se mostrarán los datos arrojados de dicha API.

La aplicación simula el inicio de sesión, que es escribir un id de usuario (lo que sea) en el respectivo formulario. Este id se guarda como una cookie de la aplicación y con ese valor un Guard valida si tiene permiso o no para acceder a la sección de Correos, en otras palabras, la sección de Correos solo podrá ser ingresada si se guarda un id de usuario sin importar qué valor sea.
